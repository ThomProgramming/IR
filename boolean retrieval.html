Kleiner Ausflug in die Aussagenlogik:¶

Fr ein beliebiges Dokument wollen wir ein Information Retrieval System implementieren das berprft, ob die Wrter Mittelmeer oder Pazifik enthalten sind. In der Aussagenlogik weisen wir beiden Aussagen Variablen zu:

  • A: fr Mittelmeer ist enthalten.
  • B: fr Pazifik ist enthalten.

Die boolsche Aussage $A$ kann dann zwei Zustnde annehmen: True oder False. Das Gegenteil von $A$ (also das Wort Mittelmeer ist nicht enthalten) wird durch $\neg A$ beschrieben. Htte $A$ also den Wert True, wre $\neg A$ False. Aussagen knnen auch verknpft werden und zwar durch Operatoren wie:

  • AND: $\wedge$ z.B. $A \wedge B$
  • OR: $\vee$ z.B. $A \vee B$

Dabei gilt:

  • $A \wedge \neg A = False$
  • $A \vee \neg A = True$

Die zuvor beschriebene Aussage, ob die Wrter Mittelmeer oder Pazifik enthalten sind, wrde also aussehen wie folgt: $$A \vee B$$

Wir knnten unser Information Retrieval System auch fragen: gib mir alle Dokumente, die Mittelmeer enthalten und wenn nicht, dann das Wort Pazifik. Dies wrde folgendermaen lauten: $$ A \vee \neg A \wedge B $$

Ein AND-Operator wird dabei stets vor einem OR-Operator evaluiert. Man kann sich dies wie beim "Punkt vor Strich" rechnen vorstellen.

Fr $A=True$ und $B=False$ ergibt die Aussage $True$: $$ A \vee \neg A \wedge B = True $$

Aufgaben Aussagenlogik¶

Teil A: Prfen Sie selbst einmal fr $A=False$ und $B=True$:

  1. $ \neg A \wedge B $
  2. $ A \wedge B \vee \neg A $
  3. $ B \wedge \neg A \vee A $

Teil B: Definieren Sie den Term fr die Aussage:

  1. Das Wrter Alpen und Meer sollen enthalten sein.
  2. Das Wort Alpen soll enthalten sein, das Wort Meer nicht.
  3. Die Wrter Alpen und Fluss oder das Wort Meer sollen enthalten sein.

Boolean Indexing¶

  • Boolean Indexing ist ein weiteres in der Veranstaltung vorgestellte Indexing-Verfahren.
  • Das Verfahren berprft fr jedes Token des Vokabulars, ob es im Dokument enthalten ist.
  • speichert das berechnete Wissen in einer Tabelle, auch als Matrix bezeichnet.

Motivation¶

  • Das Information Retrieval Problem, dass hier gelst werden soll, beruht auf der berprfung, ob ein Token in einem Dokument enthalten ist oder nicht.
  • Die Aufgabenstellung knnte also lauten: "Finde alle Dokumente, die das Wort Europa und nicht Asien enthalten".
  • Ist die Boolean Index Matrix einmal erstellt, knnen wir mittels der Aussagenlogik die Abfrage durchfhren.

Funktinsprinzip¶

Das Funktionsprinzip des Boolean Indexing ist sehr schlicht:

  1. Bilde zunchst das Vokabular $v$ des Korpus aus allen seiner Dokumente $n$.
  2. berprfe fr jedes Token $v_i$ des Vokabulars, ob es im Dokument $n_i$ enthalten ist. Wenn ja, vermerke dies mit 1, bzw True, oder 0 bzw. False wenn nicht.

Als Ergebnis erhalten wir eine Tabelle, die in den Zeilen die Dokumente enthlt und in den Spalten die Tokens des Vokabulars. Die Matrix hat also die Gre $v \times d$.

Siehe dazu auch: Manning et al., Introduction to Information Retrieval, Kapitel 1.

def create_vocab(data, n):
    tokens = set()
    doc_tokens = []
    for document in data[:n]:
        t = tokenize(document)
        tokens.update(t)
        doc_tokens.append(t)
    tokens = sorted(tokens)
    return tokens, doc_tokens

def query(input_query, index):
    res = np.array(index[input_query[0]], dtype=bool)

    for q in input_query[1:]:
        idx = np.array(index[q], dtype=bool)
        res &= idx

    idx_true = np.where(res == True)
    return index.iloc[idx_true][input_query]

def and_(q):
    def _and(index, res):
        return res & np.array(index[q], dtype=bool), q
    return _and

def or_(q):
    def _or(index, res):
        return res | np.array(index[q], dtype=bool), q
    return _or

def query2(input_query, index):
    res = np.array(index[input_query[0]], dtype=bool)
    query_words = [input_query[0]]

    for q in input_query[1:]:
        res, word = q(index, res)
        query_words.append(word)

    idx_true = np.where(res == True)

Inverted Boolean Indexing¶

  • Der inverted Index soll Problem des einfachen Boolean Indexing entgegen wirken.
  • Idee: Umkehrung des Prinzips des Boolean Indexing
    • Nun wird zu jedem Token gespeichert, in welchem Dokument es vorkommt
    • Vorteil: wir sparen uns das Vermerken ob ein Token nicht vorkommt.
    • Vorher mussten wir fr jedes vorkommende Token im Korpus fr jedes Dokument eine Aussage treffen.

Postings List¶

Das Speichern der Zuordnung eines Tokens zu den Dokumenten in denen es enthalten ist, wird in der Literatur als Posting bezeichnet. Die resultierende Liste als Posting List.

In [17]:
def create_inverted_index(data, documents):
    doc_tokens = []
    for document in data:
        t = tokenize(document)
        doc_tokens.append(set(t))

    index = pd.DataFrame()
    for i in tqdm(range(len(documents))):
        name = documents[i]
        tokens = list(doc_tokens[i])
        doc_indices = np.array([tokens, [name] * len(tokens)]).transpose()
        index = index.append(pd.DataFrame(doc_indices))
    index.reset_index()
    index.rename(columns={0: "token", 1: "document"}, inplace=True)
    return index

Dynamic Indexing¶

  • Bisher gingen wir davon aus, die Gesamtheit aller Dokumente bleibt unverndert.
  • Wird ein neues Dokument in den Korpus aufgenommen, ist es nicht im zuvor erstellten Index enthalten.
  • Suchanfragen wrde also niemals das neue Dokument in der Ergebnismenge enthalten.
  • Um nun das neue Dokument hinzuzufgen, msste der gesamte Index neu erstellt.
  • Je nach Algorithmus fr die Indexierung und Gre des Korpus schliet dies jedoch eine hohe Rechenkapazitt ein.
  • In der Praxis wird daher bei jedem neuen Dokument ein sogenannter Auxillary Index erstellt, also ein eigener Index der nur die neuen Dokumente beinhaltet.
  • Bei einer Abfrage wird die Ergebnismenge aus dem Index der des Auxillary Index hinzugefgt. In regelmigen Abstnden wird der eigentliche Index auf dem gesamten Korpus inklusive der neuen Dokumente neu gebildet und der Auxillary Index zurckgesetzt. Der Abstand sollte abhngig der Frequenz neuer Dokumente und der bentigten Dauer fr die Berechnung des Index gewhlt werden.

Auf diese Weise wird sichergestellt, dass neue Dokumente sehr zeitnah inkludiert werden knnen.
